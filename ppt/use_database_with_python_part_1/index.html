Engines

Usage

engine = create_engine('mysql://scott:tiger@localhost/test')
connection = engine.connect()
result = connection.execute("select username from users")
for row in result:
    print "username:", row['username']
connection.close()

　

  • The typical usage of create_engine() is once per particular database URL, held globally for the lifetime of a single application process.

  • A single Engine manages many individual DBAPI connections on behalf of the process and is intended to be called upon in a concurrent fashion.

  • The Engine is most efficient when created just once at the module level of an application, not per-object or per-function call.

  1. create_engine() 典型用法是每个特定的数据库 URL 调用一次，在单个应用程序的整个生命周期过程全局持久。

  2. 每个 Engine 为进程管理着很多独立的 DBAPI 连接；他会被并发地调用。

  3. Engine 每个应用程序被创建一次。

　

  • The connection is an instance of Connection, which is a proxy object for an actual DBAPI connection. The DBAPI connection is retrieved from the connection pool at the point at which Connection is created.

connection 是一个 DBAPI 连接的代理对象；它是从 pool 里获取的；

Session

  • In the most general sense, the Session establishes all conversations with the database and represents a 『holding zone』 for all the objects which you’ve loaded or associated with it during its lifespan.

  • 「Identity Map」 and 「Unit of Work」.

  • Connection & Route

  • Other

Identity Map

Ensures that each object gets loaded only once by keeping every loaded object in a map. Looks up objects using the map when referring to them.

Unit of Work

Maintains a list of objects affected by a business transaction and coordinates the writing out of changes and the resolution of concurrency problems.

Lifecycle

  1. a business transaction refers a unit of work.

  2. maintains objects

  3. commit at the end of transaction

  4. handle with concurrency problems

  5. clear resources

Usage

# an Engine, which the Session will use for connection
# resources
some_engine = create_engine('postgresql://scott:tiger@localhost/')

# create a configured "Session" class
Session = sessionmaker(bind=some_engine)

# create a Session
session = Session()

# work with sess
myobject = MyObject('foo', 'bar')
session.add(myobject)
session.commit()
session.close()

Session & Engine & Connection & Bind

  1. Sessionmaker produce a session class which contains bind information between entities and engines.

  2. A session instance use bind information to choose engines for entities.

  3. It also create/retrieve connections from chosen engines.

Session’s Problem

class User(Model):
    id = Column(Integer, primary_key=True)
    team_id = Column(Integer, ForeignKey('team.id'))


class Team(Model):
    id = Column(Integer, primary_key=True)
    leader_id = Column(Integer, ForeignKey(User.id))

Session’s Problem

user = User(1, 1)
team = Team(1, 1)
session.add_all([user, team])
session.commit()

Flush()

user = User(1, 1)
team = Team(1, 1)
session.add_all([user, team])
user.team_id = None
session.flush([user, team])
user.team_id = 1
session.commit()

Flush()

  1. Sessions produce SQL at the end of business transaction.

  2. Executed SQL does not mean written into database.

  3. Commit is the key command.
